# TeamFinder

Платформа для поиска команды и участников в IT-проекты. Разработчики могут находить проекты по своим навыкам, а владельцы проектов — приглашать подходящих специалистов.

## Возможности

- Регистрация и авторизация пользователей
- Создание и редактирование профиля (аватар, навыки, контакты)
- Публикация проектов с указанием требуемых навыков
- Поиск проектов по навыкам
- Добавление проектов в избранное
- Участие в проектах
- Просмотр участников и авторов проектов
- Фильтрация пользователей по участию в проектах

## Технологии

- **Backend**: Django 5.2, Python 3.10
- **Database**: PostgreSQL
- **Frontend**: HTML, CSS (вариант шаблонов v3)
- **Containerization**: Docker, Docker Compose
- **Libraries**: Pillow (генерация аватаров), python-decouple (настройки)

## Запуск проекта

### 1. Клонируйте репозиторий

```bash
git clone <url-репозитория>
cd team-finder-ad-main
```

### 2. Запустите виртуальное окружение

```bash
sourse venv/Scripts/activate
```

### 3. Создайте `.env`

Файл `.env` содержит конфиденциальные настройки проекта — ключ Django, параметры БД и другие переменные.

В репозитории есть пример `.env_example`, который нужно скопировать и заполнить:

```bash
cp .env_example .env
```

После этого откройте `.env` и укажите свои значения.  

| Переменная            | Назначение                                                                                                                                                 |
|-----------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **DJANGO_SECRET_KEY** | Секретный ключ Django, используемый для подписи cookie и токенов. Можно сгенерировать при помощи `get_random_secret_key` из `django.core.management.utils` |
| **DJANGO_DEBUG**      | Режим отладки. Установите `True` во время разработки.                                                                                                      |
| **POSTGRES_DB**       | Имя базы данных PostgreSQL, которую будет использовать Django.                                                                                             |
| **POSTGRES_USER**     | Имя пользователя PostgreSQL.                                                                                                                               |
| **POSTGRES_PASSWORD** | Пароль пользователя PostgreSQL.                                                                                                                            |
| **POSTGRES_HOST**     | Адрес сервера БД. В случае локальной разработки localhost.                                                                                                 |
| **POSTGRES_PORT**     | Порт подключения к БД (по умолчанию `5432`).                                                                                                               |                                                                    |
| **ALLOWED_HOSTS**     | Список хостов/доменов для доступа к сайту через запятую. Для разработки: `localhost,127.0.0.1`                                                             |

### 4. Запустите Docker (базу данных):

```bash
docker compose up -d
```

### 5. Примените миграции

```bash
python manage.py migrate
```

### 6. Создайте суперпользователя

```bash
python manage.py createsuperuser
```

### 7. Запустите сервер
```bash
python manage.py runserver
```

### 8. Откройте в браузере: http://localhost:8000
